## CAPABILITY

Kukode is a curated directory and showcase platform where **makers** submit production-grade websites/design-systems/ Astro templates, **visitors** browse, vote, and comment, and **admins** moderate submissions and manage flags. After this session, all core flows (auth, submit, vote, comment, payment, search, admin moderation, analytics) are end-to-end wired to the Turso (libSQL) database and deployed on Cloudflare Workers.

## CONSTRAINTS

### Fixed Policy (immutable)

1. **Auth is server-side sessions only.** No JWT, no client-side tokens. Session cookie is opaque; session state lives in Turso `sessions` table.
2. **One vote per user per product.** Toggle on/off only â€” no upvote-only, no weighted voting. UNIQUE constraint `(user_id, site_id)` + `db.transaction()` prevents double-vote.
3. **Role model is three-tier**: `user` â†’ `maker` (auto-upgrade on first submit) â†’ `admin` (DB-only, no UI promote path). No demote path exists.
4. **Email is transactional only.** Via Resend. No email queue, no retry beyond `Effect.forkDaemon`.
5. **Payments are Xendit invoice-only.** No subscription billing beyond one-time invoices. Team plan subscription uses `/api/store/subscribe` (same Xendit flow).
6. **Content is English-only.** i18n covers UI strings (`t('key')`) only â€” no translated content collections.
7. **Feature flags are DB-driven** (`feature_flags` table) with optional `FEATURE_*` env overrides. Admin toggle UI exists. Flags resolve via `src/lib/flags.ts`.

### Architecture Invariants

1. **DB client is module-level.** `src/db/index.ts` creates the Turso client at import time via `import.meta.env`. For Cloudflare Workers, env vars must be compiled-in via `wrangler.toml [vars]`, NOT via `wrangler secret put` (secrets are runtime-only, module-level code can't read them).
2. **Effect layer manages all DB access.** No direct `db.*` calls outside of repository live implementations. `createAppRuntime(Astro.locals)` creates a fresh `ManagedRuntime` per request.
3. **Astro SSR sends HTML + React islands.** DisqusClone (comment thread) uses `client:only="react"` â€” no SSR, rendered entirely client-side. CmdK (command palette) is vanilla TS, no framework dep.
4. **Pages that require Turso DB have error fallbacks.** Homepage (`ProductOfTheDay`, `SitesPreview`) and `leaderboard` gracefully degrade with try/catch + empty arrays â€” so DB failure doesn't crash the entire site.
5. **No CI/CD pipeline exists.** Deploy is manual via `npm run build && wrangler deploy`. Secrets must be re-set after each clean deploy.
6. **i18n is 5 locales.** `en` (default), `id` (production-grade), `bjn`, `jv`, `ms` (approximate â€” needs native-speaker review). Missing keys fall back to English.

### Security Boundaries

1. **Cron endpoint** (`/api/cron/calculate-potd`) checks `Authorization: Bearer <CRON_SECRET>` â€” but if `CRON_SECRET` is unset (dev), it passes silently. In dev, any caller can trigger POTD calc.
2. **Admin routes** (`/admin/*`) are gated by middleware `profile.role === 'admin'` â€” no exception.
3. **Sign-out** is POST-only (GET removed in API cleanup). CSRF protection via SameSite cookie.
4. **Rate limiting** via Upstash: auth endpoints per-IP, authenticated actions per-user. 429 redirects to error page for forms, returns JSON for API endpoints.

|---|---|---|
| **Visitor** | Unauthenticated user | Browse `/`, `/sites/*`, `/blog/*`, `/pricing`, `/about`; search (CmdK); read comments |

## NON-GOALS

This capability document does not cover:

- **Search indexing** â€” existing Fuse.js search is client-side over static data. No full-text search engine (Meilisearch, Algolia, Typesense) is integrated.
- **Email marketing** â€” Resend is transactional only. No campaign management, templates, or unsubscribe flow beyond DB removal.
- **CDN / edge caching** â€” `cache-control` headers are not explicitly set on any SSR page. Cloudflare edge cache is not configured.
- **CI/CD** â€” No GitHub Actions, no preview deployments, no automated testing in pipeline.
- **Performance budgets** â€” No load testing, no Core Web Vitals monitoring, no bundle size tracking.

## OPEN QUESTIONS

1. **Team plan pricing.** $9/month no-ops subscription is implemented but not realistic. Need a real pricing model before public launch.
2. **CRON_SECRET management.** Currently no secure way to rotate the cron secret. `wrangler secret` values aren't readable after set.
3. **`@libsql/client/web` in Workers.** The `/web` import works but the package has `workerd` export conditions. Should investigate if `@libsql/client` (no subpath) resolves correctly in Workers to avoid future breakage.
4. **No admin promote UI.** Promoting a user to admin requires raw SQL. Should this have an admin panel UI?
5. **No user deletion.** Profiles can't be deleted via UI. `ON DELETE CASCADE` would cascade-delete all owned data, but no endpoint exposes this.
